# プロンプト改善成果物の妥当性検証チェックリスト

## 検証の目的
作成された全ての改善版プロンプトファイル（Updated-*.md）が、元の要求に整合し、OpenAIのプロンプトエンジニアリング原則に準拠しているかを高精度に検証する。

## 検証チェックリスト

### 1. **要求仕様との整合性**
- [ ] 元のファイルと1対1でUpdated-{元のファイル名}が作成されているか
- [ ] プロンプト文字列（```text ブロック内）のみが改善されているか
- [ ] 元の文書構造（見出し、セクション、説明文）が維持されているか
- [ ] プロンプト以外の箇所は誤字脱字の修正のみか

### 2. **OpenAI 6つの戦略の適用度**
- [ ] 明確な指示：抽象的表現が具体的なアクションに置き換えられているか
- [ ] 参照テキストの提供：必要なデータソースやドキュメントが明示されているか
- [ ] タスク分割：大きなタスクがステップバイステップに分解されているか
- [ ] 思考時間の提供：「計画→実行→確認」のプロセスが明示されているか
- [ ] 外部ツールの活用：推奨ツールや技術が具体的に示されているか
- [ ] 体系的テスト：検証チェックリストや品質基準が含まれているか

### 3. **プロンプト品質の向上度**
- [ ] 具体例やテンプレートが追加されているか（コード、スクリプト、図表）
- [ ] 出力形式が明確に指定されているか（Markdown、Mermaid、JSON等）
- [ ] 評価基準が定量化されているか（5段階評価、スコアリング等）
- [ ] 自己レビュー用のチェックリストが含まれているか

### 4. **実行可能性と再現性**
- [ ] プロンプトの指示が曖昧さなく実行可能か
- [ ] 誰が実行しても同等の結果が得られる構造か
- [ ] 必要な前提条件やコンテキストが明示されているか
- [ ] エラーケースや例外処理の指針が含まれているか

### 5. **コンテキスト保持と一貫性**
- [ ] 元のファイルの意図や目的が維持されているか
- [ ] 用語の使用が一貫しているか（日本語と英語の混在が適切か）
- [ ] 前後のセクションやステップとの整合性があるか
- [ ] ドメイン固有の専門用語が正しく使われているか

### 6. **文書としての完成度**
- [ ] 日本語の文法、表現が適切か
- [ ] 段落構造が論理的で読みやすいか
- [ ] コードブロックやテーブルの記法が正しいか
- [ ] リンクや参照が有効か

### 7. **ベストプラクティスの適用**
- [ ] DDD、マイクロサービス、Clean Architectureなどの原則が適切に反映されているか
- [ ] Azure Well-Architected Frameworkの5つの柱が考慮されているか
- [ ] セキュリティ、スケーラビリティ、保守性の観点が含まれているか
- [ ] 業界標準やMicrosoft推奨事項に準拠しているか
