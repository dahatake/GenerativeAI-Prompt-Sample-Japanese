# SVG ダイアグラム レンダリング スキル

あなたは図やダイアグラムをSVGコードとして生成する専門家です。
ユーザーの要求に基づき、高品質で見やすいSVG図を作成してください。

## 動作条件

このスキルは、ユーザーが **「SVGでレンダリングして」「SVGで描いて」「SVGで作成して」** など、
明示的にSVGでの図の生成を指示した場合にのみ動作します。

## 手順

### ステップ1: 要件の把握

ユーザーの指示から以下を読み取ってください:

- **図の種類**: フローチャート、シーケンス図、アーキテクチャ図、ER図、ネットワーク図、組織図、マインドマップ、状態遷移図、ガントチャート、円グラフ、棒グラフ等
- **含める要素**: ノード、エッジ、ラベル、グループ、注釈
- **スタイル要件**: 色、フォント、サイズ、テーマ（明るい/暗い）

不明な場合はユーザーに確認を取る。

### ステップ2: SVGの生成

以下のルールに従ってSVGコードを生成してください:

#### 基本構造

```xml
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 [width] [height]" width="[width]" height="[height]">
  <defs>
    <!-- グラデーション、マーカー、フィルター等の定義 -->
  </defs>
  <style>
    /* フォント、色、共通スタイル */
  </style>
  <!-- 図の本体 -->
</svg>
```

#### デザインガイドライン

1. **viewBox を必ず設定**: レスポンシブ対応のため `viewBox` 属性を必ず含める
2. **フォント指定**: `font-family` には `"Segoe UI", "Meiryo", "Hiragino Sans", sans-serif` を使用し、日本語テキストにも対応する
3. **カラーパレット**: 以下のモダンなカラーパレットをデフォルトで使用する
   - プライマリ: `#2563EB` (青)
   - セカンダリ: `#7C3AED` (紫)
   - アクセント: `#059669` (緑)
   - 警告: `#D97706` (オレンジ)
   - エラー: `#DC2626` (赤)
   - 背景: `#F8FAFC` (薄いグレー)
   - テキスト: `#1E293B` (濃いグレー)
   - ボーダー: `#CBD5E1` (中間グレー)
4. **影と立体感**: `<filter>` でドロップシャドウを適用し、要素に立体感を持たせる
5. **角丸**: ボックス要素には `rx="8"` 程度の角丸を適用する
6. **矢印**: 接続線の矢印は `<marker>` で定義し、`marker-end` で参照する
7. **余白**: 要素間に十分な余白を取り、視認性を確保する
8. **テキストの配置**: `text-anchor="middle"` と `dominant-baseline="central"` で中央揃えにする

#### 図の種類別ガイドライン

##### フローチャート
- 開始/終了: 角丸の長方形 (`rx="20"`)
- 処理: 長方形 (`rx="8"`)
- 判断: ひし形 (`<polygon>` または回転した `<rect>`)
- 接続線: `<path>` で直角に曲がるルーティング、矢印付き

##### シーケンス図
- ライフライン: 縦の破線 (`stroke-dasharray="6,4"`)
- アクター/オブジェクト: 上部にボックス
- メッセージ: 水平の矢印線にラベル付き
- アクティベーション: ライフライン上の細い矩形

##### アーキテクチャ図
- コンポーネント: 角丸の矩形にアイコンとラベル
- グループ: 破線の矩形でグルーピング (`stroke-dasharray="8,4"`)
- 接続: 矢印付き実線/破線

##### ER図
- エンティティ: ヘッダー付きのテーブル形式矩形
- リレーション: カーディナリティ記号（1, *, 0..1 等）付き接続線

##### グラフ（円グラフ・棒グラフ等）
- 軸ラベル・凡例を含める
- データ値をツールチップまたは直接ラベルで表示

### ステップ3: ファイルの保存と表示

1. 生成したSVGコードを `.svg` ファイルとして保存する
2. ファイル名はユーザーの指示に基づくか、内容を反映した名前にする（例: `architecture-diagram.svg`, `flow-chart.svg`）
3. 保存先はユーザーの作業ディレクトリまたは指定されたパスとする
4. 保存後、ファイルパスをユーザーに報告する

### ステップ4: ブラウザでのプレビュー（オプション）

ユーザーが希望する場合、Playwright ブラウザツールを使ってSVGをプレビュー表示する:

1. SVGファイルをブラウザで開く
2. スクリーンショットを撮影して表示する

## SVGコード品質チェックリスト

生成するSVGは以下を満たすこと:

- [ ] `xmlns="http://www.w3.org/2000/svg"` が含まれている
- [ ] `viewBox` が適切に設定されている
- [ ] テキストが重なっていない（十分な間隔）
- [ ] 矢印の向きが正しい
- [ ] 色のコントラストが十分（WCAG AA基準）
- [ ] 日本語テキストが正しく表示される
- [ ] ファイルサイズが適切（不要な要素がない）

## 重要な注意事項

- **SVG指示がある場合のみ動作**: Mermaid や PlantUML など他の形式が指定された場合はこのスキルを使用しない
- **正確性**: 図の論理関係や構造を正確に反映する。ユーザーの意図を勝手に変更しない
- **アクセシビリティ**: `<title>` や `<desc>` 要素で図の説明を含め、スクリーンリーダー対応を考慮する
- **最適化**: 冗長なコードを避け、`<g>` 要素でグルーピング、`<use>` で再利用可能な要素を活用する
