# タスクの振り返り

# ツール

Microsoft 365 Copilotの**リサーチツール**を使う事を強く推奨します。


## 1. 自分のタスクの業務分類

メールやチャットなどのやり取りを分析して、どのようなタスクが発生しているのかを業務フローの観点で分類します。

Promot:
```text

あなたは、社内業務フローとタスク構造の分析担当です。

メール、チャット、会議メモ、議事録、カレンダー、添付資料、社内文書など、参照可能な社内データを横断して、業務上のやり取り・意思決定・作業の流れを分析してください。

単なる出来事の要約ではなく、実際に発生しているタスクを分類し、業務プロセス上の課題、改善余地、自動化・標準化の可能性を明らかにしてください。

## 3. タスク分類体系

抽出した各タスクは、以下の分類軸で整理してください。

### 3.1 業務フェーズ分類

各タスクを、最も近い業務フェーズに分類してください。

| 分類 | 説明 |
|---|---|
| 受付・発生 | 依頼、問い合わせ、課題、要望、アラートなどが発生した段階 |
| 情報収集 | 必要情報の確認、資料確認、関係者へのヒアリング |
| 検討・判断 | 方針検討、対応方針の比較、影響確認 |
| 作成・実行 | 資料作成、設定変更、作業実施、成果物作成 |
| 確認・レビュー | 内容確認、品質確認、レビュー、修正依頼 |
| 承認・意思決定 | 承認、決裁、Go/No-Go判断、優先順位判断 |
| 連携・共有 | 関係者への連絡、共有、引き継ぎ、通知 |
| 完了・納品 | 成果物提出、対応完了、クローズ |
| フォローアップ | 追加対応、結果確認、再発防止、運用定着 |
| 例外対応 | トラブル、差し戻し、緊急対応、想定外対応 |

### 3.2 タスク種別分類

各タスクを、業務上の性質に基づいて分類してください。

| 分類 | 説明 |
|---|---|
| 依頼対応 | 他者からの依頼に基づく対応 |
| 問い合わせ対応 | 質問、確認依頼、相談への回答 |
| 情報確認 | 必要情報、事実、状況、条件の確認 |
| 資料作成 | 文書、スライド、表、レポートなどの作成 |
| データ処理 | 集計、抽出、整形、分析、照合 |
| システム操作 | ツール操作、設定変更、登録、更新 |
| 調整 | 日程調整、関係者調整、優先順位調整 |
| レビュー | 内容確認、品質確認、フィードバック |
| 承認 | 上長・責任者による承認、決裁 |
| 報告 | 進捗、結果、課題、リスクの報告 |
| 意思決定 | 方針、対応、優先順位、採否の決定 |
| トラブル対応 | 障害、遅延、ミス、認識齟齬への対応 |
| 定型作業 | 繰り返し発生する標準的な作業 |
| 非定型作業 | 都度判断や個別対応が必要な作業 |

### 3.3 コミュニケーション分類

各タスクに関連するコミュニケーションを分類してください。

| 分類 | 説明 |
|---|---|
| 依頼 | 作業、確認、判断、対応を依頼している |
| 確認 | 事実、状況、条件、認識を確認している |
| 相談 | 判断や方針について意見を求めている |
| 報告 | 状況、進捗、結果、課題を共有している |
| 承認依頼 | 意思決定者に承認を求めている |
| フィードバック | 修正、改善、懸念、指摘を伝えている |
| リマインド | 期限、未対応、確認待ちを促している |
| エスカレーション | 上位者、別部門、専門担当へ引き上げている |
| 合意形成 | 複数関係者間で認識や方針をそろえている |
| 情報共有 | 参考情報、資料、決定事項を共有している |

### 3.4 判断負荷分類

各タスクについて、人による判断の必要度を分類してください。

| 分類 | 説明 |
|---|---|
| 低 | ルール化されており、判断がほぼ不要 |
| 中 | 条件分岐や一定の確認が必要 |
| 高 | 文脈理解、関係者調整、専門判断が必要 |

### 3.5 定型度分類

各タスクについて、標準化しやすさを分類してください。

| 分類 | 説明 |
|---|---|
| 高 | 手順、入力、出力、判断基準がほぼ固定 |
| 中 | 一部は定型だが、状況に応じた調整が必要 |
| 低 | 案件ごとの個別性が高い |

### 3.6 自動化可能性分類

各タスクについて、自動化の可能性を分類してください。

| 分類 | 説明 |
|---|---|
| 高 | ルール化・テンプレート化・システム化しやすい |
| 中 | 一部自動化、下書き生成、チェック支援が可能 |
| 低 | 人による判断、交渉、責任ある意思決定が中心 |

### 3.7 リスク分類

各タスクについて、遅延・ミス・属人化が発生した場合の影響を分類してください。

| 分類 | 説明 |
|---|---|
| 高 | 顧客影響、売上影響、法務・セキュリティ影響、重大な遅延につながる |
| 中 | 部門内外の作業遅延、手戻り、品質低下につながる |
| 低 | 影響範囲が限定的 |

### 3.8 確度分類

各分析結果には、根拠の強さに応じて確度を付けてください。

| 確度 | 判断基準 |
|---|---|
| 高 | 複数の信頼できる情報源で確認できる |
| 中 | 1つ以上の明確な根拠があるが、補足確認が望ましい |
| 低 | 断片的な情報、推測、未確認情報を含む |

---

## 4. 分析手順

以下の順番で分析してください。

1. 関連するメール、チャット、会議、カレンダー、資料、社内文書を探す
2. 情報源ごとに、関連度の高い記録を抽出する
3. 時系列で主要な出来事を整理する
4. 発生したタスクを抽出する
5. 各タスクを分類体系に従って分類する
6. 業務フローをステップごとに分解する
7. 各ステップについて、以下を整理する
   - 実施内容
   - 担当者
   - 関係者
   - 入力情報
   - 出力物
   - 使用しているシステム・データソース
   - 発生しているコミュニケーション
   - 承認・確認・依頼の有無
   - 遅延・手戻り・確認待ちの有無
8. 複数データソース間で矛盾や抜けがあれば明示する
9. ボトルネック、重複作業、待ち時間を特定する
10. 改善案を優先度付きで提案する
11. 自動化・標準化の候補を整理する

## 6. 重要な制約

- 捏造は禁止です。
- 確認できない情報を事実として書かないでください。
- 根拠が見つからない場合は、「確認できませんでした」「根拠不足」「不明」と明記してください。
- 推測する場合は、必ず「推測」と明示し、根拠となる情報も併記してください。
- 一部のデータだけで早急に結論を出さず、複数の情報源を照合してから判断してください。
- データソース間で矛盾がある場合は、どちらか一方を断定せず、矛盾点として明示してください。
- 重要な結論には、根拠となる情報源を必ず付けてください。
- 根拠が弱い結論は、確度を「高・中・低」で示してください。
- 個人の責任追及ではなく、業務プロセス改善の観点で分析してください。
- 個人の能力評価ではなく、業務設計、情報流通、意思決定構造、ツール運用の課題として整理してください。
- 機密情報、個人情報、顧客情報を扱う場合は、必要最小限の記載にとどめてください。
- データにアクセスできない、または検索できない範囲がある場合は、分析範囲の制約として明記してください。
- 同じ事象について複数の解釈が可能な場合は、断定せず、可能性を分けて記載してください。
- 改善案は、根拠、期待効果、実行難易度を分けて提示してください。
```

```text
# タスク判断チェックリスト

タスクを
1) 自分がやるべきか
2) 他者に任せるべきか
3) AIに任せられるか
を判断するためのチェックリストです。

---

## **1. 責任・役割に関する判断（自分・人間が担うべき理由）**

* [ ] このタスクは自分の職務範囲に含まれているか？
* [ ] 自分が担当しないと進まない、または他に適任者がいないか？
* [ ] このタスクは “責任の所在” が明確であり、人間の判断が不可欠か？

---

## **2. 専門性・スキルに関する判断（自分 or 他者）**

* [ ] このタスクは自分の専門知識や経験が必要か？
* [ ] 他の人よりも自分の方が効率的にこなせるか？
* [ ] 人間の高度な判断・感性・倫理が必要か？

---

## **3. 成長・学習に関する判断（自分 or 他者育成）**

* [ ] このタスクは自分のスキルアップやキャリアにプラスになるか？
* [ ] 他の人に任せることで、その人の成長につながるか？
* [ ] AIに任せると学習機会を失いそうなタスクではないか？

---

## **4. 重要度・インパクトに関する判断（責任・リスク）**

* [ ] このタスクはプロジェクトやビジネスに大きな影響を与えるか？
* [ ] このタスクの失敗は重大なリスクを伴うか？
* [ ] 評価・承認・最終判断など、人間の責任が必要か？

---

## **5. 時間・リソースに関する判断（AIが代替可能か？）**

* [ ] このタスクは自分の時間を使う価値があるか？
* [ ] 他の人に任せた方が全体の効率が上がるか？
* [ ] AIが作業を代替・高速化できる部分（ドラフト作成・リサーチ・整理など）があるか？

---

## **6. 意思決定・調整に関する判断（人間である必要性の有無）**

* [ ] このタスクには意思決定や方針策定が含まれているか？
* [ ] 関係者との調整や交渉が必要で、自分がキーパーソンか？
* [ ] AIでは担えない “意図の解釈” や “合意形成” が必要か？

---

AIが担えるかを判断するためのチェックポイントです。

### **A. AI化しやすいタスクか？**

* [ ] 情報整理、要約、文章作成、構造化など “定型化しやすい作業” を含んでいる
* [ ] ドキュメントの初稿作成（ドラフト）としてAIを使える
* [ ] 調査・比較・アイデア出しをAIに依頼できる

### **B. AIの判断で問題がない領域か？**
```
